📘 Requests and Responses - Learning Notes

Date: 08-07-2026

What are HTTP Requests and Responses?

HTTP communication follows a Client-Server model.

The client (browser) sends a Request asking for a resource.
The server processes the request and sends back a Response.

Every webpage, image, API call, or file download follows this process.

Why are Requests and Responses Important?

They allow clients and servers to communicate over the web.

Examples:

Opening a website
Logging into an account
Searching on Google
Uploading files
Calling REST APIs

HTTP Request

An HTTP Request is sent by the client to ask the server for a resource or perform an action.

Request Structure
GET / HTTP/1.1
Host: example.com
User-Agent: Chrome

Components
Request Line
HTTP Method (GET, POST, PUT, DELETE)
Resource Path
HTTP Version
Headers
Additional information about the request
Blank Line
Indicates the end of headers
Body (Optional)
Sent mainly with POST/PUT requests
Example Request
POST /login HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded

username=harish&password=12345

Meaning:

Client wants to log in.
Login credentials are sent in the request body.
HTTP Response

An HTTP Response is sent by the server after processing the request.

Response Structure
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 120

<html>
...
</html>
Components
Status Line
HTTP Version
Status Code
Status Message
Headers
Information about the response
Blank Line
Indicates the end of headers
Response Body
Requested resource (HTML, JSON, image, PDF, etc.)
Example Response
HTTP/1.1 200 OK
Content-Type: application/json

{
    "username":"Harish"
}

Meaning:

Request processed successfully.
Server returned JSON data.

Cybersecurity Notes

Understanding requests and responses is essential for:

Web application testing
API security testing
Burp Suite interception
Wireshark packet analysis
Web server log analysis
Detecting attacks like SQL Injection, XSS, CSRF, and IDOR

SOC analysts inspect requests and responses to identify suspicious behavior, unauthorized access attempts, and malicious payloads.

Key Takeaways
HTTP communication always consists of a Request followed by a Response.
Requests are initiated by the client; responses are generated by the server.
Requests contain methods, headers, and optionally a body.
Responses contain status codes, headers, and the requested resource.
Understanding this lifecycle is fundamental for networking, APIs, SOC analysis, penetration testing, and web security.
