## 💡 Concepts Learned

### Basic Network Terminology

- Host → Any device connected to a network.
- Network → Two or more hosts communicating together.
- Internet → A network of networks.
- Client → Requests a service.
- Server → Provides a service.
- Service → Function offered by the server.
- Router → Connects different networks and forwards packets.
- Wi-Fi → Wireless networking.

Memory Trick:

👤 Host → 👥 Network → 🌍 Internet

---

### Networking Features

- Packet → Small unit of data sent across a network.
- IP Address → Unique logical address of a device.
- Hostname → Human-readable device name.
- DNS → Converts hostname into an IP address.
- DHCP → Automatically assigns IP configuration.
- Ethernet → Wired network connection.
- TCP/IP → Collection of networking protocols.

---

### IPv4 & IPv6

- IPv4 → 32-bit addressing (4 octets).
- IPv6 → 128-bit addressing (8 groups).
- IPv6 was created because IPv4 addresses were running out.
- NAT allows many private devices to share one public IPv4 address.
- Porting means migrating from one technology to another (IPv4 → IPv6).

Memory Trick:

🏠 IPv4 = Small Apartment

🏙 IPv6 = Mega City

☎️ NAT = Office Reception

### Network Configuration

Two questions before configuring a network:

- Wired or Wireless?
- DHCP or Static IP?

- DHCP automatically provides:
  - IP Address
  - Subnet Mask
  - Default Gateway
  - DNS Server

- Static IP requires manual configuration.

---

### Configuration Files

- Network settings are stored in text-based configuration files.
- Linux servers often use configuration files because many run without a GUI.
- Different Linux distributions may store configuration files in different locations.

---

### IPv4 Configuration File

Important fields:

- DEVICE → Network interface
- BOOTPROTO → DHCP or Static IP
- IPADDR → IP Address
- PREFIX → Subnet Mask (CIDR)
- GATEWAY → Default Gateway
- DNS1 → DNS Server
- HWADDR → MAC Address

### IPv6 Configuration

- IPV6INIT → Enable IPv6
- IPV6ADDR → IPv6 Address
- IPV6_DEFAULTGW → IPv6 Gateway
- DHCPV6C → Obtain IPv6 automatically
- NETWORKING_IPV6 → Enable IPv6 networking

Network changes can be applied using:

- ifdown eth0
- ifup eth0

Restarting only the required interface is preferred over restarting the entire network service.

### DNS

- Linux stores DNS server information in:

  /etc/resolv.conf

- nameserver → DNS server IP address.
- host command → Queries DNS to resolve a hostname.

Example:

host google.com
