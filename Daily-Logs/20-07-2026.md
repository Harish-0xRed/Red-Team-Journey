# Daily Log - 20-07-2026

## 📚 Today's Learning

### Linux Essentials - Chapter 15 (System and User Security)

#### User Accounts and Security
- Learned why Linux uses user accounts to improve system security.
- Understood that each user has their own identity and permissions.
- Learned how file permissions control access to files and directories.
- Understood that users belong to one or more groups to simplify permission management.
- Learned the purpose of the Linux user database files:
  - `/etc/passwd`
  - `/etc/shadow`
  - `/etc/group`

#### Administrative (Root) Account
- Learned that the **root** account is the Linux administrator with unrestricted privileges.
- Understood the risks of logging in directly as the root user.
- Learned why running graphical applications (browser, email, etc.) as root is dangerous.
- Understood that forgetting to log out of the root account increases security risks.
- Learned the **Principle of Least Privilege (PoLP)**.
- Understood why Ubuntu disables direct root login by default.

#### Switching Users (`su`)
- Learned the purpose of the `su` (Switch User) command.
- Understood that `su` can switch to any user, not just the root account.
- Learned that if no username is specified, `su` switches to the root user by default.
- Learned the difference between:
  - `su`
  - `su -`
  - `su -l`
  - `su --login`
- Understood why `su -` is recommended (loads the full login environment).
- Learned that `su -` and `su - root` are equivalent.
- Learned that `su` requires the password of the target user.
- Learned to return to the original user using the `exit` command.

#### Linux Shell Prompt
- Learned the meaning of shell prompts:
  - `$` → Standard user
  - `#` → Root user
- Understood how the command prompt changes after switching users.

#### Mentor Discussion
- Compared `su` and `sudo` conceptually.
- Discussed why organizations prefer using `sudo` for administrative tasks.
- Related root privileges and user switching to SOC investigations and privilege monitoring.

## 💡 Lessons Learned

- Every Linux user should have only the permissions required for their role.
- The root account has unrestricted access and should be used only when necessary.
- Logging in directly as root increases the impact of mistakes and security vulnerabilities.
- `su -` provides a proper login shell and is recommended over plain `su`.
- The shell prompt (`$` or `#`) quickly identifies whether you are a normal user or the root user.
- Monitoring privileged account usage is an important task for SOC analysts.
