# 💡 Concepts Learned

## Linux Process Hierarchy

- Every running program is called a process.
- Every process has a unique PID (Process ID).
- Every process (except PID 1) has a Parent Process (PPID).
- Parent processes create child processes.

Example:

systemd
↓
login
↓
bash
↓
firefox

---

## PID

Process ID

Think:
"My Employee ID"

---

## PPID

Parent Process ID

Think:
"My Father's Employee ID"

# Comparison

| Command | Purpose | Memory Trick |
|---------|---------|--------------|
| ps | One-time process list | 📸 Photo |
| pstree | Parent-child relationship | 🌳 Family Tree |
| top | Live process monitoring | 🎥 CCTV |

---

# Important Observation

Both **ps** and **top** display running processes.

Difference:

- ps → Takes one snapshot and exits.
- top → Continuously refreshes CPU, RAM and process information.
