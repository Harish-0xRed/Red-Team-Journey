## 🧠 Key Concepts Learned

- Linux stores file ownership using **UID**, not usernames.
- Changing a username does **not** affect file ownership.
- Changing a UID can create orphaned files.
- Deleting a user without removing the home directory leaves orphaned files.
- `userdel -r` removes only the user's home directory and mail spool, **not** files stored elsewhere.
- Supplementary group changes require the user to log out and log back in.
- `-G` replaces supplementary groups.
- `-aG` appends supplementary groups.
- Password aging is managed using `chage`.
- User information is distributed across:
  - `/etc/passwd`
  - `/etc/shadow`
  - `/etc/group`
  - `/etc/gshadow`
