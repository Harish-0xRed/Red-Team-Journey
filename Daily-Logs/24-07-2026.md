#### 18.2 Setuid (Set User ID)

##### What is Setuid?
- Learned that **Setuid** is a special permission for executable programs.
- Understood that a setuid program runs with the **file owner's permissions**, not the permissions of the user executing it.

##### Real-World Example
- Studied why a normal user cannot directly access `/etc/shadow`.
- Learned that the **`passwd`** program has the setuid permission, allowing users to update their passwords without direct access to `/etc/shadow`.

##### Identifying Setuid
- Practiced identifying the setuid permission using:
  ```bash
  ls -l /usr/bin/passwd
  ```
- Learned that:
  - `s` = Setuid + Execute permission
  - `S` = Setuid only (Execute permission is missing)

##### Managing Setuid
- Practiced adding and removing the setuid permission.

Symbolic Method:
```bash
chmod u+s file
chmod u-s file
```

Numeric Method:
```bash
chmod 4775 file
chmod 0775 file
```

##### Four-Digit Permissions
- Learned that the first digit in a four-digit permission represents special permissions.
- Understood that:
  - `4` = Setuid
- Learned that using a three-digit permission (e.g., `755`) removes existing special permissions because Linux interprets it as `0755`.

## 💡 Lessons Learned

- Setuid applies only to executable programs.
- A setuid program temporarily runs with the file owner's privileges.
- The **`passwd`** program has the setuid permission, **not** the `/etc/shadow` file.
- Lowercase `s` indicates both setuid and execute permissions are enabled.
- Uppercase `S` indicates the setuid permission exists, but execute permission is missing.
- Three-digit permissions remove existing special permissions because Linux assumes a leading `0`.
