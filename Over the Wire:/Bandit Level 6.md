# Bandit Level 6 Notes

## Objective

Find the password for the next level by searching the entire filesystem for a file that matches specific conditions.

## Website

OverTheWire Bandit

## Level Goal

The password for the next level is stored somewhere on the server and has the following properties:

* Owned by user **bandit7**
* Owned by group **bandit6**
* Exactly **33 bytes** in size

## Commands Used

```bash
find / -user bandit7 -group bandit6 -size 33c
```

```bash
find ./ -user bandit7 -group bandit6 -size 33c
```

```bash
cat bandit7.password
```

## Command Breakdown

# find

Searches for files and directories.

# /

Starts searching from the root directory (entire filesystem).

# ./

Starts searching from the current directory.

# -user

Search for files owned by a specific user.

Example:

```bash
-user bandit7
```

# -group

Search for files belonging to a specific group.

Example:

```bash
-group bandit6
```

# -size

Search for files based on size.

Example:

```bash
-size 33c
```

`c` means **bytes**.

# cat

Displays the contents of a file.

## New Concepts Learned

### File Ownership

Every Linux file has an owner.

### File Groups

Every Linux file belongs to a group.

### Searching by File Properties

The **find** command can search files using multiple conditions such as:

* Owner
* Group
* Size

### Standard Output (stdout)

Normal command output.

### Standard Error (stderr)

Error messages generated by commands.

### stderr Redirection

Linux allows error messages to be redirected using:

```bash
2>
```

Example:

```bash
2>/dev/null
```

This sends all error messages to `/dev/null` (the Linux black hole), keeping the terminal output clean.
